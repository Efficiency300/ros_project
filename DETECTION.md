# Описание алгоритма детектирования полосы движения

## Обзор

Узел `lane_detector_node` получает изображения с камеры робота TurtleBot3, обнаруживает две жёлтые линии разметки трека и публикует нормированную ошибку бокового отклонения для PID-регулятора.

---

## Входные данные

| Параметр | Значение |
|---|---|
| Топик | `/camera/image_raw` |
| Формат пикселей | **NV21** (YUV 4:2:0, полуплоскостной) |
| Разрешение | 320×240 пикселей |
| Частота кадров | 10 кадров/с |
| QoS | Best-effort, глубина очереди = 1 (всегда берётся последний кадр) |

### Формат NV21

NV21 — это YUV-формат с субдискретизацией цвета 4:2:0. Каждый кадр хранится в памяти двумя блоками:

```
[ Y-плоскость: 320×240 байт ]
[ UV-плоскость: 320×120 байт (чередующиеся V и U) ]
Итого: 320×360 байт на кадр
```

`cv_bridge` не поддерживает NV21 напрямую, поэтому выполняется ручное декодирование:

```python
yuv = np.frombuffer(msg.data, dtype=np.uint8).reshape((msg.height * 3 // 2, msg.width))
frame = cv2.cvtColor(yuv, cv2.COLOR_YUV2BGR_NV21)
```

После декодирования изображение поворачивается на 180° (`cv2.ROTATE_180`), так как камера на роботе установлена вверх ногами.

## Конвейер обработки (13 шагов)

### Шаг 1. Масштабирование

```
resize_factor = 1.0  →  изображение уже 320×240, пропускается
```

Если `resize_factor < 1.0`, изображение уменьшается методом `INTER_LINEAR` перед всей последующей обработкой. Это снижает нагрузку на CPU.

---

### Шаг 2. CLAHE — нормализация освещения

**Метод:** CLAHE (Contrast Limited Adaptive Histogram Equalization)

Применяется только к **L-каналу** в цветовом пространстве **LAB**:

```
BGR → LAB → CLAHE(L) → LAB → BGR
```

Параметры: `clipLimit=2.0`, `tileGridSize=(8,8)`.

**Зачем:** выравнивает локальный контраст при неравномерном освещении (тени, блики), не меняя цветовые каналы. После этого жёлтый цвет ленты детектируется стабильнее.

---

### Шаг 3. Перспективное преобразование (вид сверху)

**Метод:** `cv2.getPerspectiveTransform` + `cv2.warpPerspective`

Трапециевидная область перед роботом отображается в прямоугольник на весь кадр — получается вид сверху (bird's-eye view).

```
Исходные 4 точки (в пикселях 320×240):     Целевые 4 угла:
  BL (нижний-левый)    → (0,   240)
  BR (нижний-правый)   → (320, 240)
  TR (верхний-правый)  → (320, 0)
  TL (верхний-левый)   → (0,   0)
```

Матрица преобразования **M** (3×3, однородные координаты):

$$\begin{pmatrix} x' \\ y' \\ 1 \end{pmatrix} = M \cdot \begin{pmatrix} x \\ y \\ 1 \end{pmatrix}$$

**Зачем:** после трансформации параллельные линии трека отображаются вертикальными прямыми, что значительно упрощает их поиск и аппроксимацию полиномом.

> Интерактивная калибровка: `ros2 run turtlebot3_lane_follower perspective_calibrator`

---

### Шаг 4. HSV-маска жёлтого цвета

**Метод:** `cv2.cvtColor(BGR→HSV)` + `cv2.inRange`

```
BGR → HSV → бинарная маска (0/255)
```

Диапазон по умолчанию:

| Канал | Мин | Макс | Описание |
|---|---|---|---|
| H (оттенок) | 20 | 35 | Жёлтый (OpenCV: 0–179) |
| S (насыщенность) | 100 | 255 | Исключает бледные/белые тона |
| V (яркость) | 100 | 255 | Исключает тёмные тона |

Результат: белые пиксели (255) — жёлтая лента, чёрные (0) — всё остальное.

**Зачем HSV, а не BGR:** HSV отделяет цвет (H) от яркости (V), поэтому маска остаётся стабильной при изменениях освещения.

---

### Шаг 5. Морфологическое открытие

**Метод:** `cv2.morphologyEx(MORPH_OPEN)` с прямоугольным ядром 5×5

```
Открытие = Эрозия → Расширение
```

Удаляет одиночные белые пиксели и небольшие блики, сохраняя протяжённые линии разметки.

---

### Шаг 6. Поиск начальных позиций линий

**Метод:** гистограмма нижней трети маски

```python
histogram = np.sum(mask[h*2//3 : h, :], axis=0)
left_x  = argmax(histogram[:w//2])      # левая половина
right_x = w//2 + argmax(histogram[w//2:])  # правая половина
```

В первые `cold_start_frames=5` кадров всегда используется гистограмма. Затем — позиция из предыдущего кадра (tracking).

---

### Шаг 7. Скользящие окна (Sliding Windows)

**Метод:** поиск белых пикселей в серии окон снизу вверх

```
n_windows = 10   →   10 горизонтальных полос высотой (240/10) = 24 пикселя
window_margin = 60  →  ширина окна = 120 пикселей (±60 от центра)
```

Алгоритм для каждого окна:
1. Выбрать все белые пиксели внутри окна
2. Если их `≥ min_pixels (50)` — сдвинуть центр к среднему X
3. Сохранить центроид окна `(y_center, x_center)`

Повторяется независимо для левой и правой линий.

**Зачем:** позволяет отслеживать изогнутые линии — каждое окно «следует» за линией вверх по изображению.

---

### Шаг 8. Аппроксимация полиномом 2-й степени

**Метод:** `np.polyfit(y_values, x_values, deg=2)`

По центроидам окон строится кривая `x = f(y)`:

$$x = a \cdot y^2 + b \cdot y + c$$

Условия применения нового полинома:
- суммарно белых пикселей `≥ min_lane_pixels (200)`
- валидных окон `≥ min_valid_windows (4)`

Если условия не выполнены — используется полином из предыдущего кадра (`prior`).

**Зачем x=f(y), а не y=f(x):** линии более вертикальны в bird's-eye view, функция x(y) однозначна на всём диапазоне y.

---

### Шаг 9-10. Управление prior и история ширины полосы

Если линия не детектирована `prior_expiry_frames=5` кадров подряд — prior сбрасывается.

При успешном обнаружении обеих линий измеряется ширина полосы (в пикселях) и сохраняется в скользящем буфере из 30 значений.

---

### Шаг 11. Резервный режим (одна линия)

Если одна линия полностью потеряна (prior истёк), а другая видна — недостающая синтезируется:

```python
synth_poly = known_poly.copy()
synth_poly[2] += direction * avg_lane_width   # смещение константного члена
```

`avg_lane_width` — среднее из истории (шаг 10).

---

### Шаг 12. Вычисление и публикация ошибки

**Метод:** адаптивная точка упреждения + экспоненциальное сглаживание

#### Адаптивное упреждение (lookahead)

```
base_y    = h × base_lookahead (0.6)   →  y = 144 пикс. от верха
coarse_error = (mid_x0 - w/2) / (w/2)

adaptive_y = base_y × (1.0 - 0.4 × |coarse_error|)
           ∈ [h×0.3, h×0.9]
```

При движении прямо упреждение большое (смотрит далеко). В повороте — уменьшается (реагирует на ближние точки).

#### Ошибка

$$\text{raw\_error} = \text{clip}\!\left(\frac{x_\text{mid} - w/2}{w/2},\ -1,\ +1\right)$$

- `0` — робот по центру полосы
- `+1` — сдвинут вправо
- `-1` — сдвинут влево

#### Экспоненциальное сглаживание

$$e_\text{smooth}[n] = \alpha \cdot e_\text{raw}[n] + (1 - \alpha) \cdot e_\text{smooth}[n-1]$$

`alpha_smooth = 0.7` при 10 кадр/с — сильная реакция, минимальное запаздывание.

#### Публикуемый топик

| Топик | Тип | Значение |
|---|---|---|
| `/lane/center_error` | `std_msgs/Float32` | [-1, +1] или NaN при полной потере |

NaN публикуется если оба полинома потеряны более 10 кадров подряд.

---

### Шаг 13. Отладочные изображения (только при `debug_mode=true`)

| Топик | Содержимое |
|---|---|
| `/lane/image_projected` | Bird's-eye view после warp (BGR) |
| `/lane/image_yellow_mask` | Бинарная маска жёлтого (BGR-обёртка) |
| `/lane/image_lane_debug` | Аннотированный кадр: полиномы, центроиды, маркер середины |

На отладочном изображении:
- **Синяя кривая** — левая линия (полином)
- **Красная кривая** — правая линия (полином)
- **Оранжевые точки** — центроиды левых окон
- **Голубые точки** — центроиды правых окон
- **Зелёный круг** — вычисленная середина полосы на lookahead-расстоянии
- **Жёлтый круг** — идеальный центр кадра
- **Пунктир** — вертикальная центральная линия

## Схема конвейера

```
/camera/image_raw (NV21 320×240 @ 10fps)
         │
         ▼
  [Декодирование NV21 → BGR]
  [Поворот на 180°]
         │
         ▼
  [1] Масштабирование (resize_factor)
         │
         ▼
  [2] CLAHE (нормализация L в LAB)
         │
         ▼
  [3] Перспективное преобразование → bird's-eye view (BGR)
         │           │
         │    /lane/image_projected  (debug)
         ▼
  [4] BGR → HSV → inRange → бинарная маска
         │           │
         │    /lane/image_yellow_mask (debug)
         ▼
  [5] Морфологическое открытие (шум)
         │
         ▼
  [6] Гистограмма / prior → начальный X левой и правой линий
         │
         ▼
  [7] Скользящие окна (×2: левая и правая)
         │
         ▼
  [8] np.polyfit → полином x=f(y) (×2)
         │
         ▼
  [9-11] Управление prior, синтез недостающей линии
         │
         ▼
  [12] Адаптивный lookahead → raw_error → сглаживание
         │
         ▼
  /lane/center_error (Float32, [-1..+1] или NaN)
         │
         ▼
  /lane/image_lane_debug (debug)
```

## Ключевые параметры калибровки

| Параметр | Влияет на |
|---|---|
| `perspective_src` | Форма трапеции для warp; калибруется интерактивно |
| `yellow_h_min/max` | Диапазон оттенка жёлтого; зависит от освещения |
| `yellow_s_min` | Нижний порог насыщенности; повысить при бликах |
| `yellow_v_min` | Нижний порог яркости; понизить при тёмном треке |
| `window_margin` | Ширина окна поиска; увеличить для крутых поворотов |
| `base_lookahead` | Дальность упреждения; уменьшить для резких поворотов |
| `alpha_smooth` | Сглаживание ошибки; увеличить при низком fps |
